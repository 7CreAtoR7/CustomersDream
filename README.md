# Customers Dream — Telegram-бот «Каталог макетов сайтов»

Telegram-бот, который показывает пользователю готовые **макеты сайтов** под разные виды бизнеса (казино, рестораны, маркетплейсы, недвижимость и пр.). Пользователь листает категории → подкатегории → макеты, лайкает понравившиеся и оставляет заявку менеджеру.

В админке (внутри того же бота) можно добавлять/удалять категории, подкатегории и макеты, прикреплять скриншоты, ссылки на Figma и цены, смотреть заявки и статистику.

---

## Возможности

### Для пользователя
- `/start` — приветствие и обязательное принятие пользовательского соглашения.
- Главное меню: каталог, избранное, связь с менеджером, соглашение, помощь.
- Каталог: 12 категорий бизнеса с подкатегориями (см. `bot/services/seed.py`).
- Просмотр макета: скриншот, описание, фичи, цена, кнопка «Открыть в Figma» (если ссылка задана).
- ❤️ Лайки — сохраняются в БД, доступны в «Мои избранные».
- 💬 Заявка по конкретному макету или общий контакт с менеджером.
- Уведомление админов о новой заявке.

### Для админа
- `/admin` — открывает админ-меню (только для user_id из `ADMIN_IDS`).
- Добавление/удаление категорий, подкатегорий, макетов через FSM-визард.
- Загрузка скриншотов прямо в Telegram — хранится `file_id`, никаких внешних файлов.
- Поддержка опциональной ссылки на Figma по каждому макету.
- Цена в формате `499 USD` или «по запросу».
- Список заявок, закрытие заявок, статистика.

### Заготовка под оплату
- `bot/services/payments.py` содержит каркас под:
  - **Telegram Payments** (Visa/MasterCard) — нужен `PAYMENTS_PROVIDER_TOKEN` от @BotFather.
  - **Криптовалюта (USDT TRC20)** — простой вариант (показ адреса) и план под NowPayments/CryptoBot.
- Сейчас оплата — *по запросу через менеджера*, что максимально гибко на старте.

### Безопасность и юридическая часть
- Перед использованием бот показывает **пользовательское соглашение** (`bot/utils/texts.py::AGREEMENT_TEXT` и `AGREEMENT.md`).
- В соглашении явно прописано, что **пользователь сам несёт ответственность** за выбор тематики продукта (включая чувствительные ниши: гемблинг, крипта и т.п.) — за лицензии, налоги, законность.
- Факт принятия соглашения сохраняется в БД (`users.agreed_to_terms`, `users.agreed_at`).

## Установка и запуск

> Требуется Python 3.11+

1. **Создать виртуальное окружение и установить зависимости**

   ```bash
   python -m venv .venv
   # Windows PowerShell
   .\.venv\Scripts\Activate.ps1
   # macOS/Linux
   source .venv/bin/activate

   pip install -r requirements.txt
   ```

2. **Получить токен бота**

   - Написать [@BotFather](https://t.me/BotFather) → `/newbot` → получить токен.
   - Узнать свой Telegram user_id у [@userinfobot](https://t.me/userinfobot).

3. **Сконфигурировать `.env`**

   ```bash
   cp .env.example .env   # на Windows: copy .env.example .env
   ```

   Заполнить как минимум:
   ```env
   BOT_TOKEN=123456789:AAA...
   ADMIN_IDS=123456789
   MANAGER_USERNAME=your_manager_telegram_username
   ```

4. **Запустить бота**

   ```bash
   python main.py
   ```

   При первом запуске:
   - создастся `data/bot.db`,
   - в БД появятся 12 категорий бизнеса с подкатегориями (без макетов).

5. **Открыть бота в Telegram**, нажать `/start`, принять соглашение.
   Если ваш ID указан в `ADMIN_IDS` — наберите `/admin`, и можно начинать наполнять каталог макетами.

## Как добавить макет (admin flow)

1. `/admin` → «➕ Макет»
2. Выбрать категорию → подкатегорию.
3. Ввести **название** (например, «Грузинский ресторан №1»).
4. Описание (или «Пропустить»).
5. **Прислать фото** скриншота макета (обычной картинкой в чат) или «Пропустить».
6. Ссылка на Figma (необязательно) — будет кнопкой «🔗 Открыть в Figma» под макетом.
7. Цена: `499 USD` или «Пропустить» (тогда у пользователя — *«стоимость по запросу»*).
8. Список фич построчно:
   ```
   Адаптивный дизайн
   Корзина и оплата картой
   Личный кабинет клиента
   ```
9. Готово. Макет тут же появляется в каталоге у пользователей.

## Скриншоты vs Figma

В боте поддерживается **гибридный подход**:

- **Основной показ — скриншот** (фото из Telegram). Это:
  - быстро (фото показывается сразу в чате),
  - безопасно (нет риска, что пользователь скопирует исходники Figma),
  - удобно (Telegram сам кеширует превью, бот хранит только `file_id`).
- **Опционально** к каждому макету можно прикрепить **ссылку на Figma** (View only). Под макетом появляется кнопка «🔗 Открыть в Figma» — для тех клиентов, которые хотят покликать прототип.

Если для конкретной ниши хочется скрыть исходники — просто не указывай Figma-ссылку, и пользователь увидит только скриншот.
